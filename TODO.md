## План: единственная управляющая сессия с таймаутом, “забрать управление”

### Сервер / API
- [x] Ввести сессионный ключ (UUID) и обязательный заголовок `X-TM-Session` для управляющих запросов; чужой токен → 403 “control locked”.
- [x] GET `/api/v2/session` выдаёт токен и статусы, закрепляет контроллер при первой команде/пинге.
- [x] Таймаут управления `control_timeout_sec` (флаг `--control-timeout`; 0 — не отдавать), хранение last_seen, обновление на пинге и управляющих командах.
- [x] `POST /api/v2/session/claim` — перехват при пустом/просроченном контроллере.
- [x] Для читающих эндпойнтов оставить свободный доступ; опционально добавить в `/api/v2/ws/state` информацию о контроллере.

### Клиент (UI)
- [x] При загрузке: GET `/api/v2/session`, сохраняем токен (localStorage + headers), keepalive, блокировка управления при 403.
- [x] Кнопка “Забрать управление” с POST `/api/v2/session/claim`, отображение предупреждения/lock в “Управление”, отключение управляющих контролов.
- [x] Перезагрузка восстанавливает токен, read-only при 403.

### Тесты
- [x] Go: кейсы две сессии (403 для второй), claim после таймаута, keepalive продлевает, RequireControl в базовых командах.
- [x] Playwright: две вкладки — первая контролирует, вторая видит блокировку; claim после таймаута; проверка, что кнопки управления disabled при lock.
